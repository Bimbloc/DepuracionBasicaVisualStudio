¿Para qué sirve depurar? :

La depuración es el proceso de identificar y corregir errores (bugs) en el código de software.

¿Cual es la diferencia entre depurar y ejecutar? :

El modo de depuración implica ejecutar una aplicación con un depurador adjunto, proporcionando características adicionales para la inspección de código, como los puntos de interrupción, análisis de variables…

Herramientas que podemos usar para hacer debug:

  • Visual Studio Debugger (F5 para empezar el debug o el botón  )
  • Breakpoints (Puntos de interrupción), de los cuales podemos encontrar distintos tipos:

Breakpoints en código (F9 o haciendo click al margen del código): La depuración del código se detiene al llegar al punto de interrupción, a partir de lo cual puedes analizar el código paso a paso.

         

Breakpoint condicional (Alt + F9, C o sobre un punto de ejecución, click derecho y abrir la ventana condiciones): Son útiles cuando se quiere controlar cuando y donde se ejecuta un breakpoint, estableciendo condiciones.

Expresiones condicionales(Cuando escoges condiciones condicionales, puedes elegir entre dos elecciones: Es cierto o Cuando cambie.

Es cierto: Para hacer interrupción cuando una expresión se satisfaga.

Cuando cambie: Para hacer interrupción cuando el valor de una variable cambia.

Condición de número de llamadas (Cuando escoges condiciones en la configuración de puntos de interrupción, seleccionas “Hit Count”): Útil para cuando queremos analizar, por ejemplo, una posición específica de un bucle.

Condición de filtro (Cuando escoges condiciones en la configuración de puntos de interrupción, seleccionas “Filtro”): Útil para restringir un punto de interrupción para que pare en ciertos procesos.

Puntos condicionales de funciones (Ctrl + K, B o Nuevo punto de interrupción > Punto condicional de función): Parar la ejecución del código cuando se llama a una función, útil para localizar funciones que no encuentras en el código.

Punto de interrupción de dependencia (Click derecho en un breakpoint y seleccionamos punto de dependencia): Son puntos que se ejecutan únicamente cuando se ha alcanzado un punto de interrupción previo.

Punto de interrupción temporal: (Click derecho en un breakpoint y seleccionamos punto temporal o F9 + Shift +Alt, T): Útiles cuando queremos depurar el código solo una vez.        

Puntero de ejecución: Nos muestra la línea exacta de código que se va a ejecutar, en el margen derecho. Podemos arrastrar esta flecha al punto de código que queramos analizar después.

Depuración paso a paso: Empleadas para avanzar en la depuración según los requisitos del usuario

Por instrucción(F11, o menu Debug > paso a paso por instrucción, botón menu): Avanza a la siguiente instrucción a ejecutar del código.

Por procedimientos(F10, menú Debug > paso a paso por procedimientos, botón menu): Si la línea actual contiene una llamada a una función, ejecuta el código y suspende la ejecución de la primera linea de código después de que se devuelva la función a la que se ha llamado.

Ejecutar hasta click(Botón ): Ejecuta el código hasta la linea donde se ha pulsado el botón.

  • Editor de código activo: Posibilidad de editar código durante la depuración
  • Inspeccionar variables con información sobre datos: Para conocer el valor actual de una variable, basta con colocar sobre ella el cursor, y mostrará su valor, o en caso de tener propiedades, se podrá expandir el objeto para verlas todas

  • Esto también se puede realizar con las ventanas Automático y Variables locales, en la primera, se ven las variables, su tipo y su valor actual, mostrando todas las variables que se usan en la línea actual o en la anterior.
  •  La ventana variables locales muestra las variables del ámbito actual
  • Inspección: Para poder realizar un seguimiento de una variable en concreto, empleamos una inspección, haciendo clic derecho sobre la variable y pulsando Agregar inspección.
